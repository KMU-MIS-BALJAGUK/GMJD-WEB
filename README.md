# 🚀 공모자들

**개발 기간**: 2025.09.01 ~ 2025.12.15 (16주)

🎯 **공모자들 – 공모전 참여를 위한 팀 매칭·협업 통합 플랫폼** 🎯

공모자들은 공모전에 참여하고 싶지만  
**팀원 모집의 어려움, 공모전 정보 탐색의 번거로움, 소통과 팀 관리의 불편함**을 겪는 참가자들을 위해  
공모전 참여 전 과정을 하나의 서비스로 통합한 플랫폼입니다.

팀원 찾기부터 실시간 소통, 공모전 정보 확인, 팀 및 모집, 지원 정보 관리까지  
공모전에 필요한 모든 과정을 한 곳에서 해결할 수 있도록 설계되었습니다.

## 📌 주요 기능

👥 **팀원 매칭**
- 스킬, 경험, 관심 분야 기반 팀원 탐색
- 원하는 조건의 팀원 검색 및 팀 지원

💬 **실시간 채팅**
- 팀 생성 시 즉시 사용 가능한 실시간 채팅
- 아이디어 공유 및 프로젝트 진행 소통 통합

📢 **공모전 정보 제공**
- 최신 공모전 정보 한눈에 확인
- 분야·관심사별 공모전 필터링 및 상세 정보 제공

🗂️ **팀 관리**
- 팀원 모집 현황 및 지원자 관리
- 팀 지원 및 승인 상태 확인

---

## 🌟 서비스 특징

1️⃣ **공모전 참여 전 과정을 하나의 플랫폼에서**
- 팀원 모집, 정보 탐색, 협업, 관리를 분산된 도구 없이 해결

2️⃣ **신뢰할 수 있는 팀 매칭 환경**
- 팀원의 스킬과 경험 정보를 사전에 확인 가능

3️⃣ **소통 중심의 협업 구조**
- 실시간 채팅을 통해 팀 단위 커뮤니케이션 강화

## 📑 프로젝트 규칙

### FE
| **커밋 컨벤션**

| Prefix            | 커밋 내용                                                                                 |
|--------------------|------------------------------------------------------------------------------------------|
| `feat`            | 새로운 기능 추가, 컴포넌트 파일 생성                                                     |
| `fix`             | 버그 수정                                                                                |
| `docs`            | 문서 수정                                                                                |
| `style`           | 코드 formatting, 세미콜론 누락, 코드 자체의 변경이 없는 경우 (엔터 처서 한줄 비우는 경우) |
| `refactor`        | 코드 리팩토링                                                                            |
| `test`            | 테스트 코드, 리팩토링 테스트 코드 추가, 더미 데이터 사용시                                |
| `chore`           | 패키지 매니저 수정, 그 외 기타 수정 e.g. `.gitignore`                                    |
| `design`          | CSS 등 사용자 UI 디자인 변경 (tailwind 사용시)                                           |
| `types`           | 타입 및 인터페이스 정의                                                                  |
| `comment`         | 필요한 주석 추가 및 변경                                                                 |
| `rename`          | 파일 또는 폴더명을 수정하거나 파일 위치 옮기는 작업만인 경우                             |
| `delete`          | 파일을 삭제하는 작업만 수행한 경우                                                       |
| `!BREAKING CHANGE`| 커다란 API 변경의 경우                                                                   |
| `!HOTFIX`         | 급하게 치명적인 버그를 고쳐야 하는 경우                                                  |
| `revert`          | 원래 코드로 되돌리는 작업                                                                |
